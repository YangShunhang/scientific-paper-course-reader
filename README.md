# 科研论文课程式精读 Skill

一个面向科研论文学习场景的Skill，用于将完整论文 PDF 重组为由浅入深的“课程式精读”。建议使用语言模型网页端。

它不是简单摘要工具，也不是逐段翻译工具，而是帮助刚进入某个研究方向的研究生理解：

- 论文为什么研究这个问题；
- 作者如何定义研究对象、系统、任务或假设；
- 方法建立在什么模型或问题设定之上；
- 关键公式、图片和实验应该如何读；
- 论文真正创新在哪里；
- 哪些结论可靠，哪些地方仍有局限。

---

## 适用场景

适合用于：

- 精读算法、系统、理论、实验、综述、应用或交叉学科论文；
- 帮助刚入门的研究生快速建立论文理解框架；
- 将论文转换为课程讲义式结构；
- 解释论文中的重点公式、图表、实验和创新点；
- 区分“已有技术”和“论文真正贡献”；
- 做论文汇报、组会准备、文献阅读笔记或研究选题分析。

默认输入是一篇完整科研论文 PDF。

---

## 设计目标

本 Skill 的核心目标是：

> 按照论文的科学逻辑重新组织内容，让读者先知道“为什么”，再理解“研究什么”，接着理解“如何建模和求解”，最后判断“实验是否支持结论”。

默认读者是刚进入该领域的研究生：

- 具备本科数学和基础编程能力；
- 不默认熟悉论文所在细分方向；
- 需要解释关键术语、建模逻辑、公式直觉和实验设计；
- 不需要把论文逐句翻译成中文。

---

## 核心特点

### 1. 自适应课程结构

Skill 不固定输出八节，也不机械照搬论文原章节。

它会根据论文类型、难度、篇幅和信息密度，自适应组织为若干节课程。

常见结构包括：

```text
问题背景 → 基础概念 → 系统/问题/假设建模 → 核心方法 → 重点公式/图表 → 实验设计 → 实验结果 → 创新与局限
```

如果论文内容较长或复杂，Skill 会先讲前半部分，并在当前部分结束时给出阶段性小结，等待用户继续指令。

---

### 2. 先给阅读路线，再展开讲解

正式精读前，Skill 会先给出一个简单目录：

- 论文类型；
- 难度判断；
- 推荐节数；
- 是否建议分两次讲；
- 课程目录。

这样读者可以先建立全局地图，再进入细节。

---

### 3. 公式讲解嵌入论文逻辑

本 Skill 不会把所有公式集中堆在一起讲，也不会默认所有论文的核心都是损失函数。

公式会按照论文逻辑出现的位置进行讲解，例如：

- 背景之后讲系统、问题或假设建模；
- 方法之前讲任务定义、目标或约束；
- 方法内部讲核心计算、训练或更新公式；
- 实验部分讲评价指标、提升率或统计方法；
- 理论论文中讲主定理、关键引理或复杂度界。

公式类型只是后台判断辅助，不是固定输出栏目。

可能识别的公式类型包括：

- 建模公式；
- 问题定义公式；
- 约束公式；
- 目标函数、损失函数或奖励函数；
- 算法更新公式；
- 评价指标公式；
- 理论公式。

如果论文没有某类公式，Skill 不会强行添加。

---

### 4. 系统建模不强制数学化

很多论文的方法依赖某个系统、场景、任务或假设，但不一定用公式明确表达。

因此，Skill 会根据论文实际内容解释：

- 研究对象是什么；
- 输入、输出、状态、实体或样本是什么；
- 哪些是已知条件，哪些是待求解对象；
- 方法为什么能够建立在这个设定上；
- 作者是否隐含了某些强假设。

如果建模是文字，就解释文字设定；如果是图，就解释系统结构和数据流；如果是公式，就解释变量关系和公式作用。

---

### 5. 每节结构轻量化

每节默认采用轻量课程结构：

```markdown
## 第 X 节：标题

**本节要解决的问题**  
...

**关键概念**  
...

**核心解释**  
...

**公式、图或实验讲解**  
...

**本节小结**  
...
```

没有公式、图片或实验时，对应部分会自动省略，不会为了模板强行填充。

---

### 6. 保留最终复盘表

当完整论文讲完后，Skill 会给出：

- 重点公式总表；
- 重点图片总表；
- 实验结果总表；
- 真正创新；
- 局限与不足；
- 可复现与改进建议；
- 一句话知识地图。

这些内容用于帮助读者复盘整篇论文，而不是替代正文中的逻辑讲解。

- 哪些公式应该略讲。

## 使用方式

上传一篇论文 PDF 后，可以直接说：

```text
请使用科研论文课程式精读 Skill 讲解这篇论文。
```

或者：

```text
请把这篇论文按课程方式精读，先给我一个简单目录，然后开始讲解。
```

如果希望控制输出长度，可以说：

```text
请用标准版讲解，不要太臃肿，只讲重点公式、关键图片和主要实验。
```

如果论文很复杂，也可以分段使用：

```text
先讲前半部分，重点讲问题背景、建模和核心方法。
```

继续时可以说：

```text
继续讲后半部分，包括实验、创新、局限和复现建议。
```

## 适合的论文类型

| 论文类型 | 支持情况 | 说明 |
|---|---|---|
| 算法型论文 | 支持 | 重点讲问题定义、方法框架、训练或推理公式、实验对比 |
| 系统型论文 | 支持 | 重点讲场景、系统架构、数据流、实现和性能评测 |
| 理论型论文 | 支持 | 重点讲定义、假设、主定理、证明路线和适用边界 |
| 实验型论文 | 支持 | 重点讲实验假设、设计、统计方法、结果和偏差 |
| 综述型论文 | 支持 | 重点按主题重组，而不是逐章翻译 |
| 应用型论文 | 支持 | 重点讲应用问题、方法匹配性和落地限制 |
| 混合型论文 | 支持 | 根据主线决定先讲系统、算法、理论还是实验 |

## 输出原则

Skill 遵循以下原则：

1. **课程化，而不是翻译化**  
   重新组织论文，而不是逐段翻译原文。

2. **逻辑优先，而不是格式优先**  
   先讲清楚“为什么这样设计”，再讲公式和实验。

3. **重点优先，而不是全面堆叠**  
   只展开对理解论文主线有帮助的公式、图片和实验。

4. **按需解释，而不是强行补齐**  
   论文没有某类内容时，不虚构、不强行添加。

5. **批判阅读，而不是只复述作者结论**  
   检查假设、基线、公平性、消融、复现和结论外推。

## 局限

本 Skill 主要用于帮助理解论文，不替代专业审稿或严格复现。

它可能受到以下因素影响：

- PDF 解析质量；
- 论文中公式、图片或表格是否清晰；
- 原文是否完整；
- 论文是否省略关键实现细节；
- 结果是否缺少代码、数据或统计显著性信息。

对于公式密集、证明复杂或实验结果特别多的论文，建议分多轮讲解。
