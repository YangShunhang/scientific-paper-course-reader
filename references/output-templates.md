# 输出模板

## 启动目录模板

正式精读前，先输出：

```markdown
我会把这篇论文按课程方式讲解。先给出阅读路线。

**论文类型**：...
**难度判断**：...
**推荐节数**：...
**是否建议分两次讲**：...

**课程目录**
1. ...
2. ...
3. ...
```

目录要简洁，不要在这里展开大量解释。

## 标准版每节模板

```markdown
## 第 X 节：标题

**本节要解决的问题**  
...

**关键概念**  
- **英文名（缩写，中文名）**：一句通俗解释；说明它在本文中的作用。

**核心解释**  
...

**公式、图或实验讲解**  
...

**本节小结**  
...
```

如果没有关键概念、公式、图或实验，对应部分可以省略。

## 分段输出模板

如果论文过长或复杂，先讲前半部分。结尾使用：

```markdown
本次先讲到这里。到目前为止，你已经理解了：

1. ...
2. ...
3. ...

下一部分建议继续讲：

- 核心实验设计；
- 主要结果和消融；
- 创新与局限；
- 复现和改进建议。
```

不要承诺后台继续完成。等待用户继续指令。

## 重点公式总表

全文讲完后输出：

```markdown
| 公式/编号 | 所在部分 | 作用 | 核心直觉 | 是否影响论文主线 |
|---|---|---|---|---|
| ... | ... | ... | ... | 是/一般 |
```

只收录重点公式。

## 重点图片总表

```markdown
| 图号 | 所在部分 | 内容 | 读图重点 | 与方法/实验的关系 |
|---|---|---|---|---|
| ... | ... | ... | ... | ... |
```

只收录关键图。

## 实验结果总表

```markdown
| 实验/指标 | 比较对象 | 主要结果 | 相对最佳基线 | 解释 | 可信度注意点 |
|---|---|---:|---:|---|---|
| ... | ... | ... | ... | ... | ... |
```

如果论文没有足够数值，不要编造。说明“论文未明确报告”。

```markdown

**真正创新**

1. ...
2. ...

**局限与不足**
1. ...
2. ...

**复现难点**
1. ...
2. ...

**可改进方向**
1. ...
2. ...
```

## 最终知识地图

用一句话或一条链路表达：

```markdown
问题 → 建模/假设 → 方法 → 训练/推理 → 实验 → 结论与局限
```

示例：

```markdown
这篇论文的知识地图是：它从 ... 问题出发，先把 ... 建模为 ...，再用 ... 方法解决 ...，最后通过 ... 实验证明 ...，但仍受 ... 限制。
```

## 长度控制

默认标准版应控制篇幅：

- 每节不写无关背景；
- 概念解释只解释当前需要的；
- 公式只讲重点公式；
- 图只讲关键图；
- 实验表格只总结主要结果；
- 批判性分析要具体，但不要泛泛列清单。

如果用户要求“详细讲第 X 节”，再展开推导、图解和补充背景。
