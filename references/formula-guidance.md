# 公式讲解指引

## 公式选择原则

不要试图讲完所有公式。只选择对论文主线有明显帮助的公式。

优先级通常为：

1. 让读者理解研究对象或问题设定的公式；
2. 定义输入、输出、变量、状态、任务或可行域的公式；
3. 说明目标函数、损失函数、奖励函数或优化目标的公式；
4. 限制资源、条件、概率、物理规律、安全性或可行性的约束公式；
5. 解释核心方法如何计算、更新、训练、推理或决策的公式；
6. 支撑实验结论的评价指标公式；
7. 理论型论文中的主定理、关键引理、复杂度界或收敛条件。

这些只是判断辅助。不要在输出中强行列出所有类别。

## 公式类型识别

### 1. 建模公式

用于定义研究对象、系统结构、变量关系、数据生成过程、状态空间、实体关系或基本假设。

讲解重点：

- 这个公式把现实中的什么对象抽象成数学对象；
- 哪些变量是已知输入，哪些变量是待求量；
- 这个建模为后续方法提供了什么基础；
- 是否存在强假设或简化。

适合放在“系统/问题/假设建模”相关章节中讲。

### 2. 问题定义公式

用于说明论文到底要求解什么问题，包括任务定义、优化变量、目标、可行域、输入输出映射等。

讲解重点：

- 论文要从什么输入得到什么输出；
- 决策变量或学习对象是什么；
- 为什么这个问题难；
- 后续方法如何围绕这个定义展开。

适合放在建模之后、方法之前。

### 3. 约束公式

用于限制容量、资源、预算、时间、概率、安全性、稳定性、物理条件、通信、能耗或其他现实条件。

讲解重点：

- 约束来自现实还是算法设计；
- 约束限制了哪些变量；
- 如果没有该约束，问题会发生什么变化；
- 该约束是否现实，是否过强或过弱。

只有论文真实存在并影响方法时才展开。

### 4. 目标函数、损失函数或奖励函数

用于说明模型、算法或策略在优化什么。

讲解重点：

- 目标是最大化还是最小化；
- 每一项分别鼓励或惩罚什么；
- 是否存在权重系数或超参数；
- 不同项之间是否有权衡；
- 它和训练、推理、决策或实验指标是否一致。

不要默认所有论文的核心公式都是损失函数。

### 5. 算法更新公式

用于说明参数、状态、策略、表示、估计量或解如何迭代变化。

讲解重点：

- 更新前是什么，更新后是什么；
- 每一步使用了哪些信息；
- 公式对应算法流程中的哪一步；
- 是否涉及梯度、采样、动态规划、贝叶斯更新、消息传递、注意力、递推或优化求解；
- 更新是否有收敛、稳定性或计算成本问题。

### 6. 评价指标公式

用于说明实验如何衡量性能、误差、效率、成本、鲁棒性或提升。

讲解重点：

- 指标衡量什么；
- 越大越好还是越小越好；
- 是否和训练目标一致；
- 是否可能掩盖其他问题；
- 相对提升是相对哪个基线计算的。

提升率通常写为：

```math
提升率 = \frac{新方法 - 基线}{基线} \times 100\%
```

必须说明相对哪个基线。

### 7. 理论公式

用于表达定理、证明、界、复杂度、收敛性、泛化误差或概率保证。

讲解重点：

- 结论成立需要哪些条件；
- 公式说明了上界、下界、等价关系还是概率保证；
- 对实际方法有什么意义；
- 条件是否现实；
- 理论结论和实验是否一致。

## 单个重点公式讲解模板

对重点公式，使用自然语言嵌入正文，不必机械列小标题。至少覆盖：

1. 公式在这里为什么出现；
2. 关键符号分别表示什么；
3. 每一项起什么作用；
4. 它和前文建模、当前方法或后文实验的关系；
5. 关键参数变化时，直觉上会发生什么；
6. 初学者最容易误解什么。

## 略讲公式

以下公式通常不需要展开成完整讲解：

- 常见数学定义；
- 与主线无关的记号整理；
- 重复出现的同类公式；
- 标准模块的常规公式，除非论文对其做了关键修改；
- 实验中简单平均、求和、归一化公式，除非指标本身是论文重点。

略讲时可以用一句话说明作用即可。

## 公式讲解的位置

按论文逻辑插入公式：

- 背景之后：讲问题设定或系统建模公式；
- 方法之前：讲任务定义、目标或约束；
- 方法内部：讲核心计算或更新公式；
- 实验部分：讲评价指标或提升率；
- 理论部分：讲定理、界或证明关键步骤。

不要把所有公式放到最后统一讲。最后的“重点公式总表”只用于复盘。
