# 科研论文课程式精读

## 目标

把完整科研论文 PDF 重组为适合刚入门研究生学习的课程式精读。不要机械翻译原文章节，不要固定八节。根据论文类型、难度、篇幅和信息密度，自动组织为清晰的学习路线。

默认读者：刚进入该领域的研究生，具备本科数学和基础编程能力，但不默认熟悉该论文所在细分方向的术语、模型和常见方法。

## 总体原则

1. 先给简单目录，再展开讲解。
2. 按论文逻辑推进：问题背景 → 研究对象/建模/假设 → 问题定义 → 方法 → 公式/图按需插入 → 实验 → 创新、局限与改进。
3. 公式不集中堆叠。只在它服务于当前理解进度时讲解。
4. 不强行寻找不存在的公式、图片或实验。没有就省略。
5. 不为了格式填充内容。只解释对理解论文主线有帮助的信息。
6. 保持课程感，避免无关堆砌；但对论文主线、核心贡献、关键模块、关键公式和决定实验结论的内容必须充分展开。

## 默认工作流

### 1. 读完整论文 PDF

先整体浏览论文，识别：

- 论文题目与研究领域；
- 论文类型：算法型、理论型、系统型、实验型、综述型、应用型或混合型；
- 主要问题、研究缺口和贡献；
- 方法是否依赖明确的系统、数据、理论或实验设定；
- 关键公式、关键图片、关键实验和核心结论；
- 论文难度和信息密度。

需要论文类型判断时，参考 `references/paper-type-routing.md`。

### 2. 先输出简单阅读目录

正式讲解前，先给出一个简短阅读路线：

- 论文类型；
- 难度判断；
- 推荐节数；
- 是否建议分两次讲；
- 课程目录。

不要在目录阶段展开所有细节。

### 3. 决定一次讲完还是分段讲

由智能体自行判断，不要求用户确认。

- 如果论文内容适中，可以一次性讲完。
- 如果论文很长、公式密集、实验复杂或多模块并行，先讲前半部分，在结尾说明后半部分可继续讲。
- 分段时也要保证当前部分自洽，不要在关键推理链中突然中断。

### 4. 按课程结构展开

每节使用基础课程结构，但根据内容重要性自适应展开。不要把所有章节写成同样长度：核心贡献、关键模块、关键公式、关键图表和决定实验结论的内容应明显更详细；支撑性内容适度讲清；边缘内容简要带过。

```markdown
## 第 X 节：标题

**本节要解决的问题**  
用 1–2 句话说明本节帮助读者理解什么。

**关键概念**  
只解释本节必须用到、且初学者可能不熟悉的术语。

**核心解释**  
按照因果关系讲解，不逐句翻译论文原文。

**公式、图或实验讲解**  
仅当本节中存在重要公式、关键图片或实验设计时展开。没有则省略。

**本节小结**  
用 2–4 句话总结本节最应该记住的内容。
```

如果本节属于论文核心方法、核心模块、关键训练机制、关键增强策略、关键实验或主要贡献，不能只做摘要式说明。应先判断本节在论文主线中的作用，再参考以下角度选择真正相关的部分展开：

- 作者为什么需要这个设计；
- 它解决了前文哪个具体缺口或困难；
- 它的输入、输出和中间过程是什么；
- 它和前一个模块、后一个模块如何连接；
- 它相比直接方案、已有方法或普通 baseline 的差异在哪里；
- 如果去掉它，论文的方法链条、实验结论或贡献论证会缺什么；
- 作者是否通过消融、图表或实验支持了它；
- 初学者最容易误解什么。

这些角度是讲解参考，不是硬性清单。不要机械逐条套用，也不要为了套模板引入论文中不存在的问题。

更多输出模板参考 `references/output-templates.md`。

## 公式讲解策略

公式讲解必须嵌入论文进度中。不要把所有公式集中放到一个独立章节里讲。

选择公式时遵循：

1. **主线相关**：只讲对理解问题、建模、方法、训练、推理、实验或结论有帮助的重点公式。
2. **逻辑顺序**：按照论文前后推理链讲。先解释前置建模，再解释基于建模产生的目标、约束、算法和指标。
3. **按需分类**：公式类别只是内部判断辅助，不是固定输出栏目。
4. **不凑公式**：普通定义、常识变形、重复推导或不影响理解的公式可以略讲或不讲。

需要详细公式分类和讲解方法时，参考 `references/formula-guidance.md`。

## 系统、问题与假设建模

在进入核心方法前，必须判断论文是否定义了系统、场景、数据、任务、假设或变量关系。

建模不一定是公式。可能是文字、图、变量表、问题定义、实验设定或隐含假设。

如果有明确建模，要让读者明白：

- 论文研究的现实对象是什么；
- 输入、输出、状态、实体、变量或样本分别是什么；
- 哪些是已知条件，哪些是待求解对象；
- 方法为什么能建立在这个设定上；
- 这个设定有哪些限制或隐含假设。

如果作者没有清晰建模，应指出其默认依赖的假设，不要替作者编造不存在的模型。

## 图表讲解策略

只讲关键图片和图表。重点包括：

- 系统/架构图；
- 方法流程图；
- 模型结构图；
- 关键结果图；
- 消融、敏感性或对比实验图；
- 能帮助理解论文主线的表格。

讲图时说明：

- 图中元素表示什么；
- 信息流、控制流、数据流或推理流程如何移动；
- 这张图和前后公式、方法或实验的关系；
- 初学者读图时最应该抓住什么。

## 实验讲解策略

实验部分至少解释：

- 数据集、仿真环境或实验平台；
- 对比基线；
- 评价指标；
- 参数设置和场景规模；
- 主要结果；
- 消融、敏感性、扩展性或鲁棒性实验；
- 作者结论是否被实验充分支持。

区分事实与解释：

- 事实：表格或曲线显示了什么，哪个方法更好，提升多少。
- 解释：为什么可能更好，是否有其他解释，证据是否充分。

## 创新与局限

必须区分：

- 论文真正贡献；
- 使用的已有技术；
- 针对具体问题的组合或改造；
- 证据不足、假设过强、基线不公平、只在仿真验证、缺少消融或复现困难等问题。

讲核心方法时要持续区分三层：

1. 论文直接采用的已有基础模块、常规算法或标准实验设置；
2. 作者在已有基础上的改造、组合、约束、训练方式或应用方式；
3. 这些改动为什么服务于本文问题，以及实验或消融是否支持它们确实有效。

不要等到最后创新总结时才做贡献归因；讲每个核心模块时就要说明它到底新在哪里。不要把“使用已有模块”说成“发明该模块”。

## 最终输出必须包含

如果一次性讲完整篇论文，最后提供：

1. 重点公式总表；
2. 重点图片总表；
3. 实验结果总表；
4. 真正创新；
5. 局限与不足；
6. 可复现与改进建议；
7. 一句话知识地图。

如果本次只讲前半部分，则先提供“阶段性小结”，最终总表可以等全文讲完后再给。

## 质量检查

输出前检查：

- 是否先给了简单目录；
- 是否按论文逻辑而不是原文目录机械翻译；
- 是否解释了读者进入方法前必须知道的系统、问题或假设；
- 是否只讲了重点公式；
- 公式是否嵌入对应章节，而不是堆在一起；
- 是否没有强行添加不存在的公式、图片或实验；
- 是否根据内容重要性分配篇幅：核心贡献充分展开，支撑内容适度解释，边缘内容简要带过；
- 每个核心模块是否讲清了“动机 → 机制 → 作用 → 与前后内容关系 → 证据支持或不足”；
- 是否区分已有技术与真正创新；
- 是否包含批判性分析；
- 是否适合刚入门研究生阅读。
